# Карта Маршрутов

Ниже зафиксирован первый системный проход по приложению на локальном debug-экземпляре `ssh-configurator` с типом хоста `Runtime-VM`.

## Главное

- Приложение построено как Vue 3 SPA на `createWebHashHistory()`.
- По умолчанию `/` редиректит на `/system-settings/datetime`.
- После успешного подключения диалог `Настройки SSH` закрывается, но его можно снова открыть через кнопку настроек в нижней панели.
- Для `Runtime-VM` в левом меню видны:
  - `/system-settings/datetime`
  - `/network-settings/settings`
  - `/manage/services`
  - `/keys`
  - `/diagnostics`
  - `/monitor`
- Для `Virt-Server` в коде меню предусмотрены отдельные разделы, но даже на `Runtime-VM` их маршруты можно открыть напрямую по hash-route.

## Верхнее меню Runtime-VM

### `/system-settings/*`

- `/system-settings/datetime`
  - Поля: дата, время, timezone
  - Кнопки: `Запросить`, `Установить`
  - Наблюдение: на подключённом хосте успешно показывает дату/время и timezone

- `/system-settings/kernel-isolation`
  - Поля: `Физических`, `Логических`, `Первое ядро`, `Последнее ядро`
  - Кнопки: `Запросить`, `Обновить загрузчик хоста RT`

- `/system-settings/ntp-server`
  - Поле: `IP адрес`
  - Кнопка: `Установить`

- `/system-settings/system-commands`
  - Кнопки: `Перезагрузить хост`, `Выключить хост`, `Установить`, `Запросить`
  - Опасность: `Перезагрузить` и `Выключить` являются destructive actions

### `/network-settings/*`

- `/network-settings/settings`
  - Вкладки/аккордеоны: сетевые настройки, редактирование интерфейса, создание Bond
  - Кнопки: `Применить cетевые настройки`, `Восстановить настройки по умолчанию`, `Изменить`, `Создать`
  - Для `Virt-Server` в коде дополнительно есть блок создания bridge

- `/network-settings/files`
  - Раздел файлов сетевой конфигурации
  - Кнопка: `Переименовать`

### `/manage/*`

- `/manage/services`
  - Кнопки: `Запуск VCont без службы`, `Остановить VCont`, `Создать`, `Запустить`, `Остановить`, `Перезапустить`
  - Таблица сервисов наблюдалась с именами: `vcontd`, `opcuad`, `vcont-monitor`

- `/manage/backups`
  - Поля: firmware directory, username, IP
  - Кнопки: `Обновить Vcont`, `Создать копию Vcont`, `Восстановить Vcont`

- `/manage/updates`
  - Поля: directory, username, IP
  - Кнопка: `Обновить`

### Отдельные страницы Runtime-VM

- `/keys`
  - Разделы: генерация ключей, управление ключами
  - Кнопки: `Сгенерировать ключ`, `Удалить ключи в known_hosts`, `Вывести список ключей`, `Удалить ключ`, `Добавить`

- `/diagnostics`
  - Поле выбора службы для диагностики
  - Кнопка: `Выполнить`
  - На подключённом хосте успешно выполнялся `check_status systemd-journald`

- `/monitor`
  - Два блока:
    - обновить сертификаты
    - установить агент мониторинга
  - В обоих блоках поля: directory, username, IP
  - Кнопки: `Выполнить`

## Скрытые, но доступные маршруты Virt-Server

Эти маршруты не показываются в меню на `Runtime-VM`, но router их обслуживает и страницы открываются прямым переходом:

- `/snapshots`
  - Кнопки-toolbar с title: `Вывести список`, `Создать`, `Объединить все снэпшоты`, `Удалить все снэпшоты`, `Обновить`, `Восстановить`, `Удалить`

- `/vmbackup`
  - Поля: VM name, firmware directory, username, IP
  - Кнопки: `Создать копию VM`, `Восстановить VM`

- `/manage/vm/management`
  - Кнопки-toolbar с title: `Вывести список`, `Запуск ВМ`, `На паузу`, `Возобновить`, `Перезагрузить`, `Выключить`, `Удалить из списка`, `Удалить ВМ, дисковые образы, снашоты и конфигурационные файлы`, `Консольный доступ к ВМ`, `Редактировать XML конфигурацию ВМ`

- `/manage/vm/description`
  - Поля: XML file, RAM, CPU, network type, network value, PCI, source, rename
  - Кнопки: несколько `Изменить`, `Зарегистрировать ВМ`

- `/manage/vm/xmls`
  - Кнопки: `Вывести список`, `Удалить`, `Зарегистрировать ВМ`, `Переименовать`

- `/manage/vm/images`
  - Кнопки: `Вывести список`, `Удалить`, `Переименовать`

- `/manage/vm/grouped-files`
  - Кнопка: `Вывести список`

- `/manage/vm/pci-devices`
  - Поля: PCI device, текстовые области результата
  - Кнопки: `Вывести список`, `Проверить`

## Соединение SSH

### Диалог `Настройки SSH`

Наблюдавшиеся поля:

- `Host`
- `Key file`
- `User`
- `Password`
- `Port`

Наблюдавшиеся кнопки:

- вкладки `Подключение` и `Конфигурация`
- discovery refresh
- file picker для ключа
- `Подключиться`

### Минимальный рабочий сценарий подключения

- Host: `192.168.122.10`
- Key file: `/home/ant/IdeaProjects/configurator/passgen/vcont_initial_access`
- User: `service`
- Password: пустой
- Port: `22`

После подключения наблюдалось:

- тип сервера: `Runtime-VM`
- статус: `подключен`
- маршрут мог оставаться текущим, приложение не обязательно редиректит

## Практические правила исследования

- Не запускать destructive actions без явного разрешения:
  - reboot
  - shutdown
  - reset network settings
  - массовое удаление snapshot/VM
- Для рутинной навигации использовать `goto` и `snapshot`.
- Для подключения использовать `open-settings`, `read-connection-form`, `connect-minimal`.
- Для дальнейшего углубления полезно пройти по каждому безопасному `list/request/show` действию и зафиксировать ответы таблиц и журналов.
