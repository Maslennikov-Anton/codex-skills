## Методические выводы

- Для экранов с `DataTable` и `AppTable` одного `body.innerText` недостаточно.
- Для экранов с `DataTable` и `AppTable` теперь уже есть второй слой проверки:
  - чтение строк таблиц напрямую через DOM
- Для экранов с icon-only кнопками внутри `AppAccordion` теперь есть отдельный устойчивый подход:
  - искать аккордеон по заголовку
  - кликать кнопку по индексу внутри `.content`
- Для row selection нужен осторожный подход:
  - generic `tr.click()` полезен как базовая техника
  - но для `AppTable` этого недостаточно, если нужно не только визуально выделить строку, а гарантированно изменить внутреннее Vue state страницы
  - для действий вида `select -> rename/delete/start/...` предпочтительнее page-specific helper команды, которые дополнительно проверяют изменение доступности кнопок и связанных input-полей

## Page-Specific Selection Flows

### `network-files`

- helper command: `select-network-file <fileName>`
- что подтверждает:
  - строка реально выбрана
  - `renameTarget` обновился нужным именем файла
  - кнопка `Переименовать` стала доступной
- живое подтверждение:
  - `select-network-file 9995-manual.network` -> `ok: true`
- helper command: `prepare-network-file-rename <fileName> <newName>`
- что подтверждает:
  - целевой файл выбран
  - `renameTarget` заменён на новое значение
  - кнопка `Переименовать` остаётся доступной
- живое подтверждение:
  - `prepare-network-file-rename 9995-manual.network 9995-manual-renamed.network` -> `ok: true`

### `vm/xmls`

- helper command: `select-xml-file <fileName>`
- что уже подтверждено:
  - helper корректно переходит на экран
  - выполняет `list_files_by_directory xml`
  - читает текущее state страницы после неуспешного выбора
- текущее ограничение:
  - на текущем хосте `/var/lib/libvirt/xml_store` отсутствует
  - поэтому позитивный выбор реальной строки пока не подтверждён

### `vm/images`

- helper command: `select-image-file <fileName>`
- что уже подтверждено:
  - helper корректно переходит на экран
  - выполняет `list_files_by_directory images`
  - читает текущее state страницы после неуспешного выбора
- текущее ограничение:
  - на текущем хосте `/var/lib/libvirt/images` отсутствует
  - поэтому позитивный выбор реальной строки пока не подтверждён

### `keys`

- текущее ограничение:
  - экран использует `PrimeVue DataTable`, но в текущей реализации таблица рендерится без явных текстовых ячеек строки
  - generic `select-prime-row <rowText>` здесь не помогает, потому что строка в DOM визуально пустая
- практический вывод:
  - для `Keys` сейчас надёжно работает `list_user_keys` через console evidence
  - для `delete/select` нужен page-specific обход, а не generic row selection
- рабочий обход на текущем хосте:
  - helper command: `select-first-key`
  - что подтверждает:
    - первая строка таблицы реально переходит в `p-datatable-row-selected`
    - кнопка `Удалить ключ` становится доступной
  - живое подтверждение:
    - `select-first-key` -> `ok: true`
- реализованный следующий шаг:
  - helper command: `delete-selected-key`
  - статус:
    - helper реализован
    - destructive execution в этой сессии не выполнялся
- Следующий шаг для таких экранов:
  - читать DOM строк таблиц напрямую
  - при необходимости исследовать внутренние компоненты PrimeVue / `AppTable`
  - отдельно анализировать console log после действия, если UI сам по себе лаконичен

- Для экранов с textarea safe-метод уже хороший:
  - запуск действия
  - чтение `textarea.value`

- Действия `reboot`, `shutdown`, `delete`, `reset`, `merge all`, `delete all`, `restore`, `poweroff`, `undefine`, `delete full` остаются запрещёнными без отдельного разрешения пользователя.
