# UI Quality Checklist

Используй этот reference как короткий quality gate для интерфейсов.

## Состояния

- loading обработан явно;
- empty state не выглядит как поломка;
- error state объясняет, что произошло и что делать дальше;
- success state не теряется после async действия.

## Responsive и accessibility

- Интерфейс читается на мобильной ширине.
- Навигация и критичные действия доступны с клавиатуры.
- Контраст, focus states и семантика не сломаны.

## Performance

- Нет ли лишних ререндеров или тяжелых вычислений на hot path.
- Не загружается ли больше данных, чем нужно сценарию.
- Не приводит ли анимация или визуальный эффект к ухудшению usability.
