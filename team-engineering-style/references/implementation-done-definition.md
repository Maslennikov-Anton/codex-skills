# Критерии завершения реализации

## Базовый стандарт

Изменение считается завершенным по умолчанию только после прохождения трех слоев контроля:

1. Автоматические проверки.
2. Тестовая проверка.
3. Профессиональное ревью.

## Базовый порядок

Если в проекте доступны соответствующие инструменты, использовать такой порядок:

1. Formatter.
2. Linter.
3. Type checks.
4. Релевантные тесты.
5. Профессиональное ревью.

## Уровни строгости

### Низкий риск

Примеры:
- правка текста;
- безопасный rename;
- локальный комментарий без изменения поведения.

Минимум:
- базовая самопроверка;
- formatter/linter при наличии.

### Средний риск

Примеры:
- обычная feature;
- bugfix;
- локальный refactor;
- изменение тестового слоя.

Минимум:
- formatter/linter;
- type checks при наличии;
- релевантные тесты;
- ревью.

### Высокий риск

Примеры:
- DB change;
- API change;
- общая инфраструктура;
- авторизация;
- кросс-модульный refactor.

Минимум:
- formatter/linter;
- type checks;
- релевантные и расширенные тесты;
- ревью с акцентом на регрессии, риски и полноту покрытия.

## Что проверяет ревью

Профессиональное ревью должно отвечать как минимум на вопросы:

1. Нет ли логических ошибок?
2. Нет ли вероятных регрессий?
3. Достаточно ли тестов?
4. Не появилась ли лишняя сложность?
5. Соответствует ли изменение локальным стандартам?
