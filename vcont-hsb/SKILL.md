# VCont HSB

Используй этот skill для задач вокруг горячего резервирования VCont (HSB), синхронизации между виртуальными контроллерами, конфигурации и примеров протокольных команд. Знания собраны из PDF-файлов в `/home/ant/IdeaProjects/vcont-hsb/source`.

## Рабочий процесс

1. Определи область задачи:
   - Автомат HSB-состояний, failover, heartbeat, поведение MAIN/RESERVE/STANDALONE: читай `references/hot-standby.md`.
   - Peer-to-peer TCP-канал между контроллерами и модель синхронизации данных: читай `references/synchronization.md`.
   - Значения по умолчанию `vcontcfg.json` и HSB-параметры: читай `references/config.md`.
   - Поведение `AUTH`-запроса, роли сессии или примеры XML-команд IDE: читай `references/protocol.md`.
   - Запуск VCont/eCAL в Docker, сетевой обмен между контейнерами и multicast/IPC-нюансы: читай `references/synchronization.md`, раздел "Docker и eCAL".
2. При редактировании или тестировании локальных артефактов проекта сначала изучи текущий репозиторий. Сейчас в проекте лежат PDF-документы и архивы VCont, но нет очевидного дерева исходников приложения.
3. При написании автотестов для vcont-hsb ориентируйся на уже готовый проект автотестов VCont: `/home/ant/IdeaProjects/vcont-autotests`. Сначала изучи его структуру, фикстуры, helpers, стиль тестов и команды запуска, затем адаптируй существующие подходы под HSB-сценарии.
4. Сохраняй терминологию исходных документов: VCont/ВК, PLC/PlcId, HSB_MAIN, HSB_RESERVE, HSB_STANDALONE, heartbeat, таск, луп, функциональный блок (ФБ).
5. Рассматривай факты из PDF как продуктовые/доменные требования, а не как доказательство, что текущие бинарники их реализуют. Поведение реализации проверяй отдельно, если доступны бинарники, конфиги, логи или тесты.

## Базовая модель

- Hot Standby использует два или три инстанса VCont. В каждый момент времени только один активный контроллер должен управлять процессом; остальные находятся в резерве.
- В схеме 1+1 оба контроллера собирают входные данные и выполняют пользовательскую программу, но управляющие выходы на модули ввода-вывода отправляет только активный контроллер.
- Контроллеры обмениваются heartbeat-данными: диагностикой, текущим HSB-режимом, номером итерации рабочего цикла и `PlcId`.
- `PlcId` используется как приоритет, когда нескольким контроллерам нужно детерминированно выбрать MAIN; большее значение `PlcId` побеждает.
- Сейчас синхронизируемые данные включают значения DI/DO, признаки форсирования DI/DO и состояние встроенного Modbus-сервера.
