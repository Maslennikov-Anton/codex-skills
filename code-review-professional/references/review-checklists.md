# Review Checklists

Используй этот reference как короткий чеклист перед выводом по ревью.

## Корректность

- Соответствует ли реализация заявленному поведению.
- Есть ли граничные случаи, null/empty-paths и ошибка обработки.
- Не сломан ли existing contract для API, БД, очередей, файлов или UI.

## Качество решения

- Нет ли лишней сложности по сравнению с задачей.
- Не появились ли слишком крупные функции, классы или условные ветви.
- Соблюдены ли локальные стандарты и базовые соглашения проекта.

## Проверки и доказательства

- Прогнаны ли lint, type checks и релевантные тесты.
- Есть ли достаточно evidence для изменения risky логики.
- При high-risk изменениях есть ли regression coverage или понятный план добора.
