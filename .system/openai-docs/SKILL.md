# Документация OpenAI

Давать авторитетные и актуальные рекомендации из документации разработчика OpenAI через MCP-сервер `developers.openai.com`. Для вопросов по OpenAI всегда в первую очередь использовать MCP-инструменты документации, а не `web.run`. Этот skill также может точечно загружать файлы из `references/` для вопросов по выбору моделей и запросов, связанных с актуальными моделями GPT-5, но источником истины все равно остается текущая документация OpenAI. Переходить к веб-поиску только если MCP-сервер установлен, но не возвращает полезных результатов.

## Быстрый старт

- Используй `mcp__openaiDeveloperDocs__search_openai_docs`, чтобы найти наиболее релевантные страницы документации.
- Используй `mcp__openaiDeveloperDocs__fetch_openai_doc`, чтобы получить точные разделы и корректно цитировать или пересказывать их.
- Используй `mcp__openaiDeveloperDocs__list_openai_docs` только когда нужно просматривать или находить страницы без четкого запроса.
- Загружай только релевантный файл из `references/`, если вопрос касается выбора модели или перехода на актуальные модели GPT-5.

## Краткий обзор продуктов OpenAI

1. Apps SDK: создание приложений для ChatGPT через web component UI и MCP-сервер, который предоставляет инструменты приложения ChatGPT.
2. Responses API: единая точка входа для stateful, мультимодальных и tool-using взаимодействий в агентных workflow.
3. Chat Completions API: генерация ответа модели на основе списка сообщений, составляющих диалог.
4. Codex: агент OpenAI для разработки ПО, который может писать, понимать, ревьюить и отлаживать код.
5. gpt-oss: открытые reasoning-модели OpenAI (gpt-oss-120b и gpt-oss-20b), выпущенные под лицензией Apache 2.0.
6. Realtime API: создание низколатентных мультимодальных сценариев, включая естественные разговоры speech-to-speech.
7. Agents SDK: toolkit для построения agentic-приложений, где модель использует инструменты и контекст, передает управление другим агентам, стримит частичные результаты и сохраняет полную трассу.

## Если MCP-сервер отсутствует

Если MCP-инструменты не работают или ресурсы документации OpenAI недоступны:

1. Самостоятельно выполни команду установки: `codex mcp add openaiDeveloperDocs --url https://developers.openai.com/mcp`
2. Если команда падает из-за прав или песочницы, сразу повтори ее с escalated permissions и добавь краткое обоснование для approval. Не проси пользователя запускать ее заранее.
3. Только если escalated attempt тоже не удался, попроси пользователя выполнить команду вручную.
4. Попроси пользователя перезапустить Codex.
5. После перезапуска снова выполни поиск и загрузку документации.

## Workflow

1. Уточни продуктовую область и тип запроса: общий поиск по документации, выбор модели, переход на актуальную GPT-5-модель или обновление промпта под нее.
2. Если это запрос на выбор модели, загрузи `references/latest-model.md`.
3. Если это явный запрос на переход на актуальную GPT-5-модель, загрузи подходящий reference-файл по upgrade-процессу, если он есть.
4. Если переход может требовать изменений в промпте, либо рабочий процесс исследовательский, насыщен инструментами, ориентирован на код, multi-agent или long-running, дополнительно загрузи подходящий reference-файл по prompting guide.
5. Выполни поиск по документации точным запросом.
6. Загрузи лучшую страницу и точный нужный раздел, по возможности используя `anchor`.
7. При ревью перехода на актуальную GPT-5-модель всегда явно указывай по каждому месту использования: целевую модель, стартовую рекомендацию по reasoning, оценку `phase` при необходимости, prompt blocks и статус совместимости.
8. Отвечай кратко, ссылайся на документацию и используй reference-файлы только как вспомогательный контекст.

## Карта reference-файлов

Читай только то, что действительно нужно:

- `references/latest-model.md` -> вопросы о выборе модели и формулировки вида "best/latest/current model"; перед ответом перепроверяй все рекомендации по актуальной документации OpenAI.
- `references/upgrading-to-gpt-5p4.md` -> использовать как текущий reference-файл для явных запросов про переход на GPT-5.4 и планирование такого перехода; сверяй checklist и guidance по совместимости с актуальной документацией OpenAI.
- `references/gpt-5p4-prompting-guide.md` -> использовать как текущий reference-файл для переписывания промптов и апгрейда prompt-behavior под GPT-5.4; перед ответом проверяй prompting guidance по текущей документации OpenAI.

## Правила качества

- Считай документацию OpenAI источником истины; избегай спекуляций.
- Держи цитаты короткими и в рамках policy limits; предпочитай пересказ с цитированием.
- Если разные страницы расходятся, явно укажи различие и сошлись на обе.
- Reference-файлы являются только удобными подсказками; для volatile guidance, например рекомендуемых моделей, upgrade-инструкций и prompting advice, приоритет всегда у актуальной документации OpenAI.
- Если документация не покрывает запрос пользователя, скажи об этом прямо и предложи следующие шаги.

## Заметки по инструментам

- Для вопросов по OpenAI всегда сначала используй MCP-инструменты документации.
- Если MCP-сервер установлен, но не возвращает полезных результатов, только тогда переходи к веб-поиску.
- При fallback на веб-поиск ограничивайся официальными доменами OpenAI (`developers.openai.com`, `platform.openai.com`) и обязательно приводи источники.
